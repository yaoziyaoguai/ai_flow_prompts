# ai_flow_prompts 仓库规则

本文件约束任何在此仓库中工作的 Coding Agent 的行为。

---

## 仓库定位

这是一个 **Markdown-only** 的 prompt 模板库。不包含程序代码、不包含脚本、不包含依赖管理。

## 硬性规则

### 禁止事项

- 不要加入程序代码（.py / .ts / .js / .sh / .go 等），除非用户明确授权
- 不要写入真实 secret / API key / token / 密码
- 不要写入内部 URL / 真实 IP / 真实域名（示例用 `example.com` / `your-project` 等占位符）
- 不要写入公司敏感信息、真实项目名、真实人员名
- 不要写入真实日志内容
- 不要 push / tag / 修改 remote，除非用户明确要求
- 不要引入 package manager / 依赖
- 不要读取 .env 或任何可能包含 secret 的文件

### 必须遵守

- 只维护 Markdown prompt 模板
- 修改模板前先检查结构一致性（与其他模板的格式对齐）
- 新增模板必须符合统一格式（见下文）
- 修改时优先保持高内聚、低耦合、低重复
- 不要机械拆文件——一个模板只应为一个清晰的使用场景服务
- 不要制造过度抽象——如果模板之间共享内容不超过 30%，不要提取"公共模板"
- 所有示例必须脱敏，使用 `[placeholder]` 或 `[your-xxx]` 格式
- 如果需要用户输入，必须真正使用 AskUserQuestion，并提供 Other/free-text 选项

## 模板统一格式

每个模板文件必须包含以下部分：

```markdown
# 模板名称

## 适用场景
## 不适合场景
## 可组合模块
## 完整模板
## 使用提醒
```

缺少任何部分都是不合格的模板。

## 修改时的检查清单

- [ ] 格式是否与其他模板一致？
- [ ] 是否引入了项目特定上下文？
- [ ] 是否使用了真实数据（如果是，替换为占位符）？
- [ ] 适用/不适合场景是否写清楚了？
- [ ] 可组合模块是否标注了？
- [ ] 使用提醒是否覆盖了变量替换和风险点？

## 质量原则

- 中文为主，直接、工程化、可复制
- 少鸡汤、不写营销风格、不写玄学 prompt
- 模板要像能直接发给 Coding Agent 的真实指令
- 宁可缺，不要滥——不够成熟的模板不要放进来
