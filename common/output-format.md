# Output Format —— 输出格式约束

## 适用场景

- 需要 Agent 输出结构化、可解析的结果
- 多人协作，需要统一的输出格式
- 结果需要被后续流程消费（如自动化流水线）
- 审计/评审报告需要直接可用

## 不适合场景

- 探索性对话，输出格式自然变化
- 简单问答，不需要结构化输出
- 输出是给人快速浏览的（过度的格式反而降低可读性）

## 可组合模块

- 几乎所有 `coding-agent/` 模板都可以加载此模板以约束输出
- `common/brutal-honesty.md` —— 严重程度定义由此模块提供，本模块只约束输出结构

## 完整模板

~~~text
# Output Format

请按以下格式组织你的输出。

## 结构要求

1. **Summary**：3-5 句话总结结论（放在最前面）
2. **Details**：详细分析，按优先级或类别组织
3. **Action Items**：明确的、可执行的下一步（如果有）

## 文件引用格式

引用单个位置：`[file-path]:[line-number]`

引用多个位置时使用列表：
- `[file-path]:[line-number]` —— [一句话说明]

## 表格格式

需要对比或列举时，使用 Markdown 表格：

```
| 维度 | 状态 | 说明 |
|------|------|------|
| ... | PASS / FAIL / UNKNOWN | ... |
```

## Action Items 格式

每个 action item 必须包含：
- **动作**：具体做什么
- **优先级**：CRITICAL / HIGH / MEDIUM / LOW（定义见 `common/brutal-honesty.md`）

## 不要做的事

- 不要在结论中使用模糊措辞（"可能"、"似乎"、"也许"）
- 不要在详细分析中重复 Summary 的内容
- 不要输出没有具体动作的 Action Items
- 不要混用不同级别的严重程度标记
~~~

## 使用提醒

- 此模板约束的是输出**形式**，不约束输出内容——配合 `common/scope-lock.md` 使用效果更好
- 对于简短的日常任务，不需要加载此模板
- 严重程度的语义定义见 `common/brutal-honesty.md`，本模板只约束格式
