# 安全边界说明

## 仓库安全边界

### 绝对禁止存储的内容

- API key / token / secret / password
- 数据库连接字符串（含密码）
- 内部 URL / 真实 IP 地址
- 真实项目名（如涉及非公开的商业项目）
- 真实人员姓名 / 邮箱 / 联系方式
- 真实日志内容
- .env 文件内容
- 任何可以定位到真实组织或个人的信息

### 示例数据规则

所有示例必须使用以下占位符格式：

| 要替换的内容 | 使用占位符 |
|-------------|-----------|
| 项目名 | `[your-project]` |
| 仓库路径 | `[your-repo-path]` |
| API endpoint | `[your-api-endpoint]` |
| 文件名 | `[your-filename]` |
| 模块名 | `[your-module]` |
| 函数名 | `[your-function]` |
| 数据库表名 | `[your-table]` |
| 用户/角色 | `[your-role]` |

### 引用外部项目时的规则

模板中可以引用真实的开源项目名（如 React、Django、Claude Code），但：
- 不引用非公开的内部项目
- 不引用可能暴露组织信息的项目

## 操作安全边界

### 默认禁止的操作

本仓库的 CLAUDE.md 默认禁止以下操作：

- git push / git tag
- 修改 git remote
- 读取 .env 文件
- 联网操作（除非用户明确要求）
- 读取用户私人文件

### 用户授权例外

如果用户明确要求执行上述操作，可以执行。但：
- 执行前确认用户意图
- 不主动建议执行这些操作

## 模板使用安全

### 用户应确保

- 在将模板用于真实项目时，替换所有占位符后再发送给 Agent
- 不要在 prompt 中包含敏感信息
- 审计类模板的产出物（审计报告）可能包含对代码的详细描述，注意保密

### 模板设计安全

- 所有编程类模板默认要求"先查证据再改代码"（防止盲改引入 bug）
- 所有编程类模板默认禁止 push / tag / 读 .env
- 红队/审计类模板强调"只读" + "残忍诚实"，防止破坏性操作
- SDD/实现类模板要求维护 implementation-notes.md，确保可追溯
