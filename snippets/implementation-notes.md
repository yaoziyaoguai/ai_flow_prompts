# Implementation Notes —— 实现笔记维护片段

## 适用场景

- 嵌入到任何实现类模板中（SDD、implementation-loop、fix-audit-issues 等）
- 确保实现过程可追溯
- 方便后续回顾和理解设计决策

## 完整片段

---

## 实现笔记要求

在整个实现过程中，必须维护 `[project-path]/docs/prompts/implementation-notes.md`。

### 文件作用
- 记录实现过程中的关键决策和理由
- 记录遇到的问题和解决方案
- 记录修改的文件和修改内容
- 保障后续回顾时能理解"为什么当时这样做"

### 记录格式

```markdown
# Implementation Notes —— [功能/任务名]

## [日期] 步骤 [N]：[简短标题]

### 做了什么
- [修改 1]
- [修改 2]

### 为什么这样做
- [决策理由]

### 修改的文件
- `path/to/file1` — [改动描述]
- `path/to/file2` — [改动描述]

### 遇到的问题
- [问题] → [解决方案]

### 待处理
- [遗留问题]
```

### 记录时机
- 每次完成一个独立的实现步骤后立即记录
- 不要等实现全部完成后再补写

### 记录原则
- 记录的是"决策"和"理由"，不是"我写了哪些代码"（代码 diff 已经说明了）
- 保持简洁——每条 1-3 行
- 如果这一步骤没有值得记录的决策，可以跳过

---

## 使用提醒

- 这是一个**可嵌入片段**，嵌入位置通常在实现类模板的流程说明之后
- 确保 `[project-path]` 指向实际的项目路径
- implementation-notes.md 在任务完成后不删除，归档到 `docs/prompts/history/`
